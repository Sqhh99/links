# 开题报告正文

## 1. 毕业设计（论文）选题背景、研究意义

近年来，随着互联网技术的快速发展和5G网络的全面普及，远程办公、在线教育和视频会议已成为人们日常工作生活中不可或缺的重要组成部分。据统计，全球视频会议市场规模从2019年的约140亿美元增长至2024年的超过500亿美元，年复合增长率超过20%。特别是后疫情时代，企业远程协作、在线课堂、远程医疗问诊等场景的需求急剧增长，用户对视频会议系统的实时性、稳定性和跨平台能力提出了更高的要求。然而，传统的视频会议软件在高并发场景下仍存在端到端延迟高、弱网环境下画质劣化严重、跨平台兼容性不足等突出问题。WebRTC作为一种由Google主导开发并经W3C和IETF标准化的开放实时通信技术，为解决上述问题提供了坚实的技术基础，但将其应用于实际的多端会议系统产品化开发中仍面临诸多工程挑战，如信令服务器架构设计、媒体流高效转发、音视频采集与渲染优化、多终端形态适配等技术难题亟需攻克。

本课题设计并实现一套基于C++和WebRTC技术的多端实时音视频会议系统，采用"桌面原生+Web浏览器"双轨并行的跨平台策略：桌面端基于C++17标准和Qt 6框架开发原生客户端，主要面向Windows平台（Linux平台后续适配），充分发挥原生应用在性能和系统资源调用方面的优势；移动端和macOS平台则通过Web端实现，用户可直接通过浏览器访问会议系统，兼容Android和macOS设备，降低用户安装门槛。系统核心通信能力基于LiveKit开源框架和WebRTC协议栈实现，支持多人实时音视频会议、桌面/窗口屏幕共享、即时文字消息等核心功能，并采用SFU（选择性转发单元）架构实现低延迟的多方媒体流分发。该研究对于推动实时通信技术在企业协同办公、远程教育培训、远程医疗问诊、在线技术支持等多个领域的深度应用具有重要的实践意义。同时，本课题为C++在现代Web实时通信领域的工程化应用提供了新的技术路径和可复用的参考方案，有助于促进国产化实时通信软件的自主可控发展，提升关键基础软件的技术储备。


## 2. 国内外研究现状和发展趋势

国内在基于WebRTC的实时音视频会议系统研究与开发方面取得了显著进展。在商业产品层面，腾讯会议、钉钉、飞书等国产视频会议平台已在企业协作市场占据重要地位，积累了丰富的大规模分布式系统工程实践经验，其中腾讯会议在疫情期间单日会议并发量峰值突破千万级别，展现了国内在高并发实时通信系统架构方面的技术实力。在SDK生态方面，声网Agora、融云、即构科技等国内厂商推出了成熟的商用实时通信SDK产品，覆盖音视频通话、直播连麦、互动白板等多种场景，但这些商用方案在底层核心技术自主可控方面仍有持续提升的空间。学术研究层面，国内高校和研究机构围绕WebRTC架构优化、SFU媒体服务器负载均衡、弱网环境下的自适应传输策略、音视频编解码效率提升等方向开展了大量探索性工作。然而，针对C++原生桌面客户端结合现代开源RTC框架（如LiveKit）进行系统化工程实践的研究相对较少，多数研究聚焦于Web端或移动端应用，对于如何充分发挥C++在性能密集型场景下的优势、实现高效的GPU加速渲染和低延迟采集传输等问题，尚缺乏完整的技术方案和工程参考。

国外在WebRTC和实时通信系统研究方面处于领先地位，形成了完善的技术标准体系和活跃的开源生态。WebRTC由Google于2011年开源发布，历经十余年的持续迭代，已成为W3C和IETF的正式标准规范，获得了Chrome、Firefox、Safari、Edge等主流浏览器的原生支持。在开源项目方面，Jitsi Meet提供了完整的开源视频会议解决方案，MediaSoup专注于高性能SFU服务器实现，而LiveKit作为新一代开源实时通信平台于2021年发布，以其现代化的架构设计、优秀的开发体验和多语言SDK支持（包括Go、JavaScript、Swift、Kotlin及本课题使用的C++/Rust FFI），迅速成为构建自托管RTC服务的热门开源选择。学术研究方面，国际学者在拥塞控制算法（如Google的GCC算法、BBR算法）、回声消除与噪声抑制、自适应码率控制、多路复用与Simulcast/SVC分层编码等核心技术领域持续取得突破，相关成果被快速应用于开源项目和商业产品中。

在发展趋势方面，实时音视频会议系统正朝着智能化、安全化和边缘化三个方向演进。AI增强的音视频处理能力正在成为行业标配，包括基于深度学习的实时背景替换与虚化、智能降噪（如RNNoise等开源方案）、语音增强与超分辨率重建等技术日益成熟；端到端加密（E2EE）日益受到重视以满足GDPR等隐私合规要求，Insertable Streams API等标准为浏览器端E2EE提供了技术支撑；多编码器自适应选择（H.264/VP8/VP9/AV1）和SVC分层编码技术持续优化弱网用户体验。未来，边缘计算与CDN深度融合将降低跨地域传输延迟，WebAssembly技术有望统一跨平台开发体验，与大语言模型结合的智能会议助手（实时转录、摘要生成、多语言翻译）也将获得更广泛的应用。

## 3. 主要研究内容和研究方法

本课题旨在设计并实现一套基于C++和WebRTC技术的多端实时音视频会议系统，主要功能模块如下：

**用户登录与认证模块**：负责用户身份验证和会议准入管理。该模块实现用户名输入、房间选择/创建功能，通过信令服务器获取JWT（JSON Web Token）访问令牌，确保会议安全性和用户身份的合法性。

**会议管理模块（ConferenceManager）**：作为系统核心模块，负责会议的全生命周期管理。该模块实现与LiveKit服务器的WebSocket连接管理、参会者状态同步、轨道（Track）订阅与发布、连接状态监控与自动重连机制，是音视频通信的核心调度中心。

**摄像头采集模块（CameraCapturer）**：负责本地视频的采集与处理。该模块基于Qt Multimedia框架，实现摄像头设备枚举、视频帧采集、帧率控制（支持30FPS），并将采集的视频帧转换为LiveKit SDK所需的格式进行传输。

**麦克风采集模块（MicrophoneCapturer）**：负责本地音频的采集与处理。该模块实现麦克风设备选择、音频流采集、音频格式转换（48kHz采样率、单声道16位PCM），并通过LiveKit AudioSource接口实现音频数据的实时传输。

**屏幕共享模块（ScreenCapturer）**：负责桌面或窗口的实时捕获与分享。该模块在Windows平台上采用DXGI Desktop Duplication和Windows.Graphics.Capture API实现高效屏幕捕获，支持全屏幕共享和单窗口共享两种模式，并提供屏幕/窗口选择对话框供用户选择共享目标。

**视频渲染模块（GLVideoWidget）**：负责音视频画面的高效渲染。该模块基于OpenGL ES 2.0着色器技术，实现GPU加速的视频帧渲染，支持纹理流更新、画面镜像、参与者名称叠加显示，相比传统CPU绑定的QPainter渲染大幅降低资源占用。

**即时通讯模块（ChatPanel）**：负责会议中的文字消息通信。该模块通过LiveKit DataChannel实现可靠的消息传输，支持文本消息的发送与接收、消息历史记录显示、发送者身份标识等功能。

**参会者管理模块（ParticipantItem）**：负责参会者列表的展示与状态管理。该模块实时显示参会者的加入/离开状态、音视频开关状态（麦克风/摄像头图标），并支持点击切换主视频画面。

**网络通信模块（NetworkClient）**：负责与信令服务器的HTTP/HTTPS通信。该模块实现Token请求、房间列表获取、房间创建等RESTful API调用，基于Qt Network模块实现异步网络请求处理。

**系统设置模块（SettingsDialog）**：负责系统配置的管理与持久化。该模块实现音视频设备选择（摄像头、麦克风、扬声器）、信令服务器地址配置、用户偏好设置的保存与读取，配置数据基于QSettings实现本地持久化存储。

本系统采用"桌面原生+Web浏览器"双轨并行的跨平台策略。桌面端基于C++17标准和Qt 6框架实现原生客户端应用，主要面向Windows平台（Linux平台后续适配），使用LiveKit C++ SDK（基于Rust FFI）处理WebRTC核心通信功能，充分发挥原生应用在GPU渲染、系统设备调用和资源占用方面的性能优势。移动端和macOS平台采用Web端实现，用户通过浏览器直接访问会议系统，兼容Android和macOS设备上的主流浏览器（Chrome、Safari、Firefox等），降低跨平台开发成本和用户安装门槛。信令服务器采用Go语言开发，基于Gorilla Mux路由框架和LiveKit Server SDK实现Token生成和房间管理API。系统采用模块化架构设计，核心层（core）负责业务逻辑，界面层（ui）负责用户交互，工具层（utils）提供日志记录和配置管理等通用功能。开发过程中采用CMake构建系统管理项目依赖和编译流程，使用Protobuf进行跨语言数据序列化，最终通过功能测试和性能测试验证系统的稳定性与实时性。
